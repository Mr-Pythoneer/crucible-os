One system, five modes

Not five separate OS images — one base install, switched live via distro-modectl switch <mode>.

Gaming
Proton-GE, Wine-staging, Bottles, GameMode, MangoHud. Performance governor, max GPU state.
AI
Local-first LM Studio (text + vision LLMs) + ComfyUI (image generation). No API keys, no cloud, no usage limits — models auto-sized to your GPU across six tiers.
Server
SSH (key-only), Docker, Netdata. Power-saving profile, headless-capable.
Creative
FreeCAD, Blender, DaVinci Resolve, NVENC/NVDEC-accelerated ffmpeg.
Normal
A polished, macOS-style default look — dock, theme, balanced power.

Configure your build

Crucible OS auto-detects your GPU on first boot and preloads the local AI models that fit it — so you never download a model your card can't run. Pick a GPU to preview exactly what you'd get. (Nothing downloads here; models are pulled on first boot by distro-ai-setup --install.)

VRAM is the binding constraint for local LLMs, so tiers are keyed on it. Two identical cards pool (2×48GB → 96GB); mixed vendors don't. Datacenter cards route to Server mode, not desktop AI.

Six hardware strains

Strains are a build-time hardware-class profile — what desktop environment and packages ship by default. Every strain still gets all 5 modes above; a strain just picks a sensible starting point. Selected via iso/build.sh <strain>.

workstation
Full GNOME desktop, no special tuning. The default.
laptop
Same desktop + battery/power management (tlp).
lowspec
LXQt instead of GNOME — the lightest official Ubuntu desktop.
server
No desktop environment at all. Fully headless.
handheld
Steam-Deck-class devices.scaffolded, not differentiated yet
cloud
cloud-init, no DE.delivery format still TODO

What this actually promises

No "runs every Windows app" claims — that promise isn't real anywhere, including SteamOS or CrossOver. Here's the honest split.

What's real
  • Broad game/app compatibility via Proton-GE + Wine-staging + Bottles, pre-tuned out of the box
  • Fully local AI — LM Studio (LLMs) + ComfyUI (image gen), auto-sized to your GPU across six tiers, zero forced cloud round-trips
  • Native CAD/video tools that actually work well on Linux today: FreeCAD, Blender, DaVinci Resolve
  • Nvidia + AMD drivers preinstalled, Secure Boot handled, not ignored
What's explicitly not attempted
  • Kernel-anticheat multiplayer games where the publisher blocks Linux — that's policy, not a technical gap any distro can engineer around
  • SolidWorks / AutoCAD / Premiere / After Effects under Wine — years of dedicated effort elsewhere hasn't solved this well
  • A literal macOS global app-menu / Mission Control clone — no stable GNOME equivalent exists yet

Status

Said plainly, not buried: this is design-and-scaffolding stage. The full framework — driver scripts, all 5 mode bundles, 6 hardware strains, a live-build ISO pipeline, a Calamares installer config, CI — exists and is execution-tested where possible without real hardware, but no ISO has been built yet and nothing has run on real target hardware.

The TODO.md in the repo is the live, itemized source of truth — check there before believing anything here is further along than it is.
